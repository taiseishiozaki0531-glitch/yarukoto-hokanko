# Implementation Plan

## 進め方の前提

- このタスク計画は、初心者が上から順番に進めやすいように並べる。並行実行用の `(P)` マーカーは付けない。
- 各サブタスク完了後は、原則として `npm run build` を実行して、少なくともビルドが通ることを確認する。
- Supabase の SQL、RLS policy、DB 変更は、実行前に必ず SQL 全文をユーザーへ提示し、ユーザーが「実行して」と明示するまで実行しない。Supabase MCP を使う場合も同じ扱いにする。
- 初期リリース対象外の AI 分類、通知、外部 API 連携、共有機能、画像アップロード、Google カレンダー連携、YouTube API 連携は実装タスクに含めない。
- `updated_at` は現在の design.md に従い、初期リリースでは Server Action の更新処理で現在時刻を設定する。DB trigger 方式へ変更する場合は、tasks 実行前に design.md の再確認と承認を行う。
